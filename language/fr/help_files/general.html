Aide interactive pour les paramètres Hesk

 

» Réglages généraux

Titre du site Web

Un titre court de votre site Web, par exemple "Mon joli site". Il sera utilisé comme titre du lien affiché dans le coin supérieur gauche de l'interface client pour leur permettre de revenir sur votre site Web.

URL de site web

URL de votre site Web, par exemple "http://www.example.com". Assurez-vous qu'il s'agit d'une URL valide et que vous la commencez par http:// ou https:// . Il sera utilisé comme URL du lien qui sera affiché dans le coin supérieur gauche de l'interface client et leur permettra de revenir sur votre site Web.

 

Fermer la fenêtre

 

Courriel du webmestre

Adresse e-mail par défaut du webmaster. Assurez-vous d'entrer une adresse e-mail valide, elle peut être la même que l'e-mail d'assistance.

 

Fermer la fenêtre

 

"De :" e-mail

Votre adresse e-mail "De :". Il s'agit de l'adresse e-mail qui apparaîtra comme expéditeur de l'e-mail (De : en-tête) dans tous les e-mails envoyés par HESK. Il s'agit généralement d'un e-mail que vous ne lisez pas et auquel vous ne répondez pas. Assurez-vous d'entrer une adresse e-mail valide, elle peut être la même que celle de l'assistance, mais il est recommandé d'en utiliser une autre.

"De :" nom

C'est le nom qui sera affiché comme expéditeur de l'e-mail (De :) dans tous les e-mails envoyés par HESK. Si aucun nom n'est défini, l'en-tête de l'e-mail De : n'affichera que votre adresse e-mail De :.

Thème du site

Il s'agit du thème qui sera utilisé sur toutes les pages HESK destinées aux clients. Tous les thèmes correctement installés seront affichés dans la liste déroulante. Le bouton "Tester le dossier de thèmes" vous aidera à résoudre les problèmes liés aux thèmes qui ne fonctionnent pas.

CSS d'administration

Ici, vous pouvez spécifier l'URL d'un fichier CSS personnalisé qui sera inclus dans le panneau d'administration.

Utilisez votre fichier CSS personnalisé pour écraser le style de panneau d'administration par défaut HESK ou ajouter des règles CSS supplémentaires sans modifier les fichiers source.

 

Fermer la fenêtre

 

» Paramètres de langue

Default Language

Si plusieurs langues sont installées pour Hesk, ce sera la langue par défaut. Toutes les langues correctement installées sont affichées dans la liste déroulante. Veuillez vous référer au fichier readme.html pour obtenir des instructions sur la façon d'installer plus de langues. Le lien "Dossier de langue de test" vous aidera à résoudre les problèmes d'installation de langues non fonctionnelles.

Si plusieurs langues sont activées, vous pouvez définir votre langue préférée sur la page "Profil".

Plusieurs langues

L'activation de ce paramètre permettra à vos utilisateurs et à votre personnel de choisir leur langue préférée dans la liste des langues installées. N'activez cette fonctionnalité que si vous fournissez une assistance dans toutes les langues installées !

 

Fermer la fenêtre

 

» Paramètres de la base de données

Hébergeur de base de données

Serveur hôte de base de données, "localhost" fonctionne pour la plupart des utilisateurs. Ne changez PAS cela à moins que vous ne sachiez quel est votre hôte MySQL. Demandez à votre hébergeur si vous n'êtes pas sûr ou si vous rencontrez des problèmes de connexion à la base de données MySQL.

Nom de la base de données

Le nom de votre base de données MySQL. Vous devez en créer un dans votre panneau de contrôle d'hébergement Web ou demander cette information à votre hébergeur.

Nom d'utilisateur de la base de données

Le nom d'utilisateur que vous utilisez pour vous connecter à votre base de données MySQL. Vous devez créer un utilisateur MySQL dans votre panneau de contrôle d'hébergement Web ou demander cette information à votre hébergeur

 

Fermer la fenêtre

 

Mot de passe de la base de données

Le mot de passe que vous utilisez pour vous connecter à la base de données MySQL. Vous devez créer un utilisateur MySQL dans votre panneau de contrôle d'hébergement Web ou demander cette information à votre hébergeur

Préfixe de tableau

Le préfixe qui sera ajouté à tous les noms de table. Si défini sur hesk_ la table "users" sera en fait nommée "hesk_users" dans votre base de données. Ceci est utile si vous souhaitez exécuter plusieurs installations de Hesk à l'aide d'une base de données MySQL, donnez simplement à chaque installation un nom de préfixe de table unique (hesk1_, hesk2_, hesk3_, ...)
