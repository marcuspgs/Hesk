» Paramètres du service d'assistance

Titre du service d'assistance

Intitulé de votre service d'assistance, par exemple "Service d'assistance de mon entreprise"

URL Hesk

Dossier d'administration

Nom du dossier où se trouvent les fichiers d'administration.

Le dossier doit exister dans le dossier HESK principal. Sur la plupart des systèmes, les noms sont sensibles à la casse.

Caractères autorisés (les autres seront supprimés) : a-z A-Z 0-9 _ -

Par défaut : admin

Dossier des pièces jointes

Nom du dossier dans lequel les pièces jointes seront stockées.

Le dossier doit exister dans le dossier HESK principal et être accessible en écriture par PHP. Sur la plupart des systèmes, les noms sont sensibles à la casse.

Caractères autorisés (les autres seront supprimés) : a-z A-Z 0-9 _ -

Par défaut : pièces jointes

Fermeture automatique des tickets

Si un ticket n'a aucune activité du client pendant X jours, il se ferme automatiquement. Réglez sur 0 pour désactiver la fermeture automatique.

Ouverture maximale

Un nombre maximum de tickets ouverts qu'un client peut avoir. Lorsque cette limite est atteinte, le client ne pourra plus soumettre de nouveaux tickets jusqu'à ce que ceux qui existent soient résolus. Affecte uniquement les tickets soumis via le formulaire en ligne.

Régler sur 0 pour désactiver la limitation du nombre maximal de tickets ouverts.

Afficher comme "dû bientôt"

Définissez le nombre de jours avant la date d'échéance pendant lesquels un ticket apparaîtra comme "Dû bientôt" dans la liste des tickets.

Peut être défini sur une valeur comprise entre 1 et 999 jours. Paramètre par défaut : 7 jours.

Formulaire de réponse

Masquer les réponses aux tickets

Lors de l'affichage d'un ticket avec de nombreuses réponses, Hesk peut masquer les anciennes réponses et afficher uniquement les plus récentes.

Cela permet aux membres du personnel d'afficher les dernières réponses à un ticket sans avoir à faire défiler les nombreuses anciennes.

Le personnel peut afficher les réponses masquées à tout moment en cliquant sur le lien "Afficher les réponses précédentes".

Limiter la largeur du ticket

La largeur maximale souhaitée de la partie texte d'un ticket (message et réponses).

Sélectionnez soit pleine largeur (le texte s'étendra sur toute la largeur disponible de l'écran) soit la largeur maximale du contenu en pixels.

La longueur de la ligne est un facteur important pour une lisibilité confortable, surtout si vous vous attendez à une lecture prolongée.

Autoriser la connexion automatique

Si défini sur OUI, le personnel aura la possibilité de se connecter automatiquement à son compte chaque fois qu'il ouvrira le panneau d'administration HESK à partir de son ordinateur. S'il est désactivé, HESK ne pourra mémoriser que les noms d'utilisateur, pas les mots de passe.

Attribuer automatiquement les tickets

Les tickets seront automatiquement attribués à un membre du personnel lors de leur soumission.

Plusieurs facteurs sont pris en compte lors de l'attribution automatique des tickets, tels que la catégorie de ticket et le nombre de tickets attribués actuellement ouverts.

Rouvrir les tickets

Réglé sur ON pour permettre aux clients de rouvrir un ticket une fois qu'il a été fermé. Lorsqu'il est défini sur OFF une fois fermé, le client devra ouvrir un nouveau ticket.

Note de réponse

Si cette option est activée, les clients pourront marquer les réponses du personnel comme Utiles ou Inutiles. Ces commentaires des clients seront utilisés pour évaluer le personnel.

Priorité client

Si cette option est activée, les clients pourront définir la priorité/l'urgence de leurs tickets, allant de Faible à Élevé. S'il est défini sur OFF, tous les tickets seront soumis avec la priorité par défaut (faible) et seul le personnel pourra changer le niveau de priorité pour un niveau supérieur.

Cependant, seulement le personnel pourra voir les billets par numéros séquentiels.

S'afficherait comme :
http://www.some-long-example.com/sub-folder/another-fol ... long_value

ReCaptcha

ReCaptcha est un service anti-bot gratuit de Google.

Pour utiliser ReCaptcha, vous devez :

  1. S'inscrire à ReCaptcha
  2. Cliquez sur Obtenir reCAPTCHA
  3. Entrez les détails de votre site Web dans le formulaire Enregistrer un nouveau site
  4. Copiez la clé du site dans les paramètres HESK
  5. Copiez la clé secrète dans les paramètres HESK
  6. Enregistrer les paramètres HESK
  7. Essayez de soumettre un nouveau ticket via l'interface client pour vérifier que reCAPTCHA fonctionne correctement.

Utiliser une question anti-SPAM

Active/désactive l'utilisation de la question de sécurité anti-SPAM. C'est un moyen très simple mais assez efficace de lutter contre les spammeurs (pas s'ils décident de cibler spécifiquement votre site Web, mais cela devrait bloquer tous les robots spammeurs parcourant le Web). L'idée est de créer une question unique à laquelle seuls les humains peuvent répondre. Tu peux utiliser le code HTML dans la question. Les réponses ne sont pas sensibles à la casse. Quelques exemples :

-> Question -> Réponse
De quelle couleur est l'eau? Bleu
Quel est le prochain chiffre après quatre ? (n'utilisez que des chiffres) 5
Tapez le code d'accès ABCDE ici : ABCDE
Code d'accès : (trouvez-le en bas de notre page de liens) Quelque code

Attention! NE PAS utiliser ces exemples, utilisez une question unique qui ne sera utilisée que sur votre site!

Limite de tentatives de connexion et durée d'interdiction (minutes)

Cette fonctionnalité limite les attaques par force brute pour se connecter à HESK ou afficher les tickets.

Après la limite des tentatives de connexion du nombre de tentatives infructueuses pour afficher un ticket ou se connecter à HESK, l'adresse IP sera interdite pendant durée d'interdiction (minutes).

Pour désactiver la limitation des tentatives de connexion infructueuses, définissez la valeur sur 0 (NON recommandé!).

Cadres

Activer pour empêcher le chargement de HESK dans des cadres sur des sites Web tiers. Cela permet de lutter contre les tentatives de "clickjacking".

Lorsqu'il est activé, HESK enverra un en-tête HTTP X-Frame-Options : SAMEORIGIN au navigateur.

Si vous souhaitez afficher HESK dans un cadre sur un site Web sous un nom de domaine différent de celui sous lequel HESK est installé, désactivez cette fonctionnalité.

Dans le cas ci-dessus, vous devrez peut-être également définir le paramètre Cookies sur Aucun et utiliser les connexions https://!

 

Fermer la fenêtre

 

 

 

 

 

 

Cookies

Ce paramètre contrôle la SameSite attribut des cookies HTTP. Pour une explication détaillée, veuillez consulter cet article.

Bref:

Strict - les cookies ne seront envoyés qu'aux demandes sous votre nom de domaine. Suivre un lien e-mail, par exemple, peut vous obliger à vous reconnecter.

Lax - valeur Hesk par défaut. Les cookies ne sont pas envoyés sur des sous-requêtes intersites normales (par exemple, pour charger des images ou des cadres dans un site tiers), mais sont envoyés lorsqu'un utilisateur navigue vers le site d'origine (c'est-à-dire lorsqu'il suit un lien).

None - cookies sera envoyé à toute demande, mais uniquement sous un lien (https://) . Utilisez ce paramètre si vous affichez Hesk dans un cadre sur un domaine tiers.

Si vous souhaitez afficher HESK dans un cadre sur un site Web sous un nom de domaine différent de celui sous lequel HESK est installé, définissez-le sur None et utilisez une connexion https://.

 

Fermer la fenêtre

 

 

 

 

 

 

SSL

Vous pouvez forcer HESK à utiliser des connexions sécurisées (https au lieu de http).

Pour gérer ce paramètre, ouvrez d'abord manuellement le fichier admin_settings_help_desk.php dans votre navigateur en utilisant https://, par exemple :

https://example.com/hesk/admin/admin_settings_help_desk.php

Cela permet de s'assurer que SSL est correctement configuré et fonctionne sur le serveur avant de configurer HESK pour le forcer.

Clé d'accès URL

Utilisez la clé d'accès URL pour empêcher l'accès Web non autorisé à vos fichiers cron Hesk, tels que:
- /inc/mail/hesk_imap.php,
- /inc/mail/hesk_pop3.php,
- /cron/email_overdue_tickets.php, etc.

Lorsqu'une clé d'accès est définie et que quelqu'un souhaite accéder à l'un de ces fichiers via une adresse URL (navigateur Web), il devra spécifier la clé correcte en ajoutant ?key=XXXXXXXX à la Adresse URL. Exemple :

https://example.com/hesk/inc/mail/hesk_imap.php?key=XXXXXXXX

Votre clé d'accès doit être une chaîne aléatoire d'au moins 20 caractères. Les caractères de clé d'accès valides sont: a-z A-Z 0-9 _ . -

La clé d'accès n'est pas utilisée lorsque les fichiers cron sont accessibles via l'interface de ligne de commande, uniquement lorsqu'ils sont servis via http(s).

Si vous laissez la clé d'accès à l'URL vide, elle ne sera pas requise/utilisée.

Authentification multifacteur (MFA)

Utilisez l'authentification multifacteur pour obliger les utilisateurs à s'authentifier avec leur mot de passe ainsi qu'une méthode d'authentification secondaire.

- L'activation de "Require Multi-Factor Authentication" activera l'authentification MFA pour les utilisateurs nouveaux et existants.

- La désactivation de "Require Multi-Factor Authentication" ne désactivera pas MFA pour les utilisateurs existants ; cela n'affectera que les nouveaux utilisateurs.

Cette minuterie est réinitialisée lorsqu'un utilisateur se déconnecte.

» Pièces jointes

Utiliser des pièces jointes

Sélectionnez OUI pour activer les pièces jointes. Si cette option est activée, les clients et le personnel d'assistance auront la possibilité de joindre des fichiers aux tickets d'assistance.

REMARQUE: Si et comment les pièces jointes fonctionneront en fonction des paramètres de votre serveur, veuillez consulter le fichier readme.html pour connaître les considérations et les éventuelles limitations des pièces jointes sur votre serveur.

Nombre par publication

Nombre de pièces jointes autorisées par publication. La plage optimale est d'environ 1 à 4. Ce paramètre n'est disponible que si "Utiliser les pièces jointes" est défini sur OUI.

 

Fermer la fenêtre

 

Taille de fichier maximale

Taille maximale des fichiers joints. Notez que la taille maximale du fichier de téléchargement est limitée par les paramètres de votre serveur, veuillez vous référer à readme.html pour plus d'informations sur les limites de pièces jointes. Ce paramètre n'est disponible que si "Utiliser les pièces jointes" est défini sur OUI.

Types de fichiers autorisés

Extensions de fichier autorisées pour joindre des fichiers. Pour ajouter d'autres extensions, utilisez le même formatage .ext et séparez-les par une virgule. Ce paramètre n'est disponible que si "Utiliser les pièces jointes" est défini sur OUI.
