Aide interactive pour les paramètres Hesk

 

Expéditeur de l'e-mail

Sélectionnez si HESK doit envoyer des e-mails en utilisant la fonction PHP mail() ou en utilisant un serveur SMTP configuré manuellement.

PHP mail() devrait fonctionner sur la plupart des configurations, si ce n'est pas le cas, vous pouvez essayer de configurer un serveur SMTP à la place.

Variables SMTP disponibles:

Hôte SMTP - nom d'hôte de votre serveur SMTP, généralement localhost ou mail.example.com
Port SMTP - votre port SMTP, généralement 25
Délai d'expiration SMTP - Délai d'expiration SMTP en secondes
Cryptage - activez si votre serveur SMTP nécessite des connexions cryptées
Nom d'utilisateur SMTP - votre nom d'utilisateur SMTP (si utilisé)
Mot de passe SMTP - votre mot de passe SMTP (si utilisé)

Canalisation des e-mails

Active le routage des e-mails (création de tickets à partir d'e-mails).

Cette option active simplement la fonctionnalité de canalisation dans HESK, pour une configuration complète de la canalisation de courrier électronique, veuillez vous référer au fichier readme.html.

 

Fermer la fenêtre

 

 

 

 

Récupération POP3

Ce paramètre active ou désactive la récupération POP3 - création de tickets à partir d'e-mails en se connectant à un compte de messagerie à l'aide d'une connexion POP3.

Pour obtenir des instructions sur la configuration de la récupération POP3, veuillez consulter le fichier readme.html.

Les variables disponibles sont :

Task Timeout : nombre de minutes d'attente avant qu'une tâche de récupération POP3 existante Achevée. Aucune nouvelle tâche de récupération ne démarrera tant que le délai d'attente n'aura pas été atteint. Définissez sur 0 pour désactiver les délais d'attente et toujours démarrer une nouvelle tâche de récupération.

Hôte POP3 : nom d'hôte de votre serveur POP3, généralement mail.example.com
Port POP3 : votre port POP3, généralement le 110. Gmail, par exemple, utilise le 995
Protocole TLS - activez si votre serveur POP3 nécessite le protocole TLS
Conserver une copie : si cette option est activée, HESK ne supprimera pas les messages du serveur POP3
Nom d'utilisateur POP3 - votre nom d'utilisateur POP3
Mot de passe POP3 - votre mot de passe POP3

 

Fermer la fenêtre

 

 

 

 

Récupération IMAP

Ce paramètre active ou désactive la récupération IMAP - création de tickets à partir d'e-mails en se connectant à un compte de messagerie à l'aide d'une connexion IMAP.

Pour obtenir des instructions sur la configuration de la récupération IMAP, veuillez consulter le fichier readme.html.

Les variables disponibles sont :

Task Timeout : nombre de minutes d'attente avant qu'une tâche de récupération IMAP existante Achevée. Aucune nouvelle tâche de récupération ne démarrera tant que le délai d'attente n'aura pas été atteint. Définissez sur 0 pour désactiver les délais d'attente et toujours démarrer une nouvelle tâche de récupération.

IMAP Host - nom d'hôte de votre serveur IMAP, généralement mail.example.com
Port IMAP - votre port IMAP, généralement 143 pour non chiffré et 993 pour SSL/TLS
Cryptage : définissez le cryptage IMAP requis
Conserver une copie - si activé, HESK ne supprimera pas les messages du serveur IMAP
Nom d'utilisateur IMAP - votre nom d'utilisateur IMAP
Mot de passe IMAP - votre mot de passe IMAP

 

Fermer la fenêtre

 

 

 

 

Ne pas valider les certificats de serveur

Si coché, Hesk ne validera pas les certificats du serveur TLS/SSL.

La validation du certificat doit généralement être ignorée si le serveur utilise des certificats auto-signés.

Nous vous recommandons vivement de ne pas désactiver la validation des certificats sur les serveurs de production et lors de la connexion à des serveurs tiers, car cela vous rend vulnérable aux attaques de l'homme du milieu.

Do not reply

Si coché, Hesk essaiera de détecter les e-mails envoyés depuis "Do not reply" adresses e-mail et les ignorer.

Examples:
do-not-reply@example.com
do_not_reply@example.com
no_reply@example.com

Revenu

Si coché, Hesk essaiera de détecter et d'ignorer les e-mails renvoyés.

Par exemple, il ignorera les e-mails envoyés depuis mail-deamon@example.com et les e-mails avec un objet tel que "Courrier retourné", "Non distribuable", "Absent du bureau" ou similaire.

Doublons

Si cette case est cochée, Hesk détectera et ignorera un e-mail si Hesk a déjà traité un e-mail contenant le même message dans le "Timeframe" secondes.

Lorsque vous testez la fonctionnalité e-mail vers ticket, désactivez ce paramètre ou envoyez des e-mails de test avec des messages uniques.

Boucles d'e-mails

Paramètres permettant de détecter les boucles d'e-mails créées à partir de répondeurs automatiques ou d'e-mails non distribuables.

Tout nombre d'e-mails supérieur au Accès max à partir d'une adresse e-mail dans un délai de délai secondes sera ignoré par le routage des e-mails.

Définissez Max Hits sur 0 pour désactiver la détection de boucle.

Détecter les fautes de frappe dans les e-mails

Cette fonction essaiera de détecter les fautes d'orthographe courantes des noms de domaine de messagerie et suggérera une version corrigée.

Par exemple, si le client entre user@gmial.com HESK le détectera et suggérera user@gmail.com à la place.

HESK comparera les domaines de messagerie aux domaines répertoriés dans la liste des fournisseurs de messagerie.

 

Fournisseurs de messagerie

Une liste des fournisseurs de messagerie courants utilisés par vos clients. HESK utilisera cette liste pour essayer de détecter les adresses e-mail mal saisies.

Entrez un domaine par ligne, sans le signe @.

Vous pouvez saisir n'importe quel nombre de noms de domaine courants, mais essayez de ne pas en faire trop; saisissez uniquement les plus courants.

Avertir le client quand

Sélectionnez quand HESK doit envoyer une notification par e-mail aux clients :

  • lorsque le client soumet un nouveau ticket d'assistance

    Remarque : si le ticket est soumis par e-mail (acheminement des e-mails ou récupération POP3), HESK peut ignorer l'envoi d'une nouvelle notification de ticket au client si l'objet de l'e-mail contient l'une des balises répertoriées dans la zone de texte (une balise par ligne).

    Vous pouvez l'utiliser pour éviter réponses automatiques aux e-mails détectés comme possible SPAM par le filtre anti-spam de votre serveur.
  • lorsqu'un ticket d'assistance est marqué comme Résolu (sans répondre)

 

Notez que les préférences de notification sont définies individuellement par ticket lorsque le personnel répond à un ticket ou lorsque le personnel soumet un ticket depuis l'interface d'administration.

 

Fermer la fenêtre

 

 

 

 

 

 

 

 

Supprimer la réponse citée

Cette fonction supprime les réponses citées de tous les e-mails des clients avant qu'elles ne soient converties en ticket à l'aide de la canalisation d'e-mails ou de la récupération POP3.

Tous les e-mails envoyés aux clients commenceront par cette ligne :

------ Répondre au-dessus de cette ligne ------

Tout texte sous cette ligne sera supprimé lorsqu'un e-mail sera importé dans la réponse au ticket.

 

Fermer la fenêtre

 

 

 

 

Exiger un message

Si cette option est activée, les e-mails provenant du routage des e-mails ou de la récupération POP3 seront ignorés s'ils ne contiennent aucun message.

Si désactivé, même les e-mails sans message seront transformés en tickets.

 

Fermer la fenêtre

 

 

 

 

Fichiers intégrés

Activez si vous souhaitez que HESK enregistre les fichiers d'e-mails intégrés (par exemple, les images intégrées) en tant que pièces jointes lors de l'importation de tickets via la canalisation d'e-mails ou la récupération POP3.

Les fichiers dépassant la limite maximale de pièces jointes autorisées seront ignorés.

Notez qu'avec HESK, les pièces jointes ont priorité sur les fichiers intégrés. Par exemple, si vous autorisez 2 pièces jointes et recevez un e-mail avec 1 pièce jointe et 3 images intégrées, HESK enregistrera la pièce jointe et la première image intégrée, le reste sera ignoré.

 

Fermer la fenêtre

 

 

 

 

Plusieurs e-mails

Si un client essaie de saisir plusieurs e-mails (séparés par une virgule) dans le champ E-mail lors de la soumission d'un ticket HESK n'utilisera que le premier e-mail par défaut et ignorera tous les autres e-mails.

Pour permettre aux clients de saisir plusieurs e-mails, activez cette fonctionnalité en cochant la case.

 

Fermer la fenêtre

 

 

 

 

Confirmer l'e-mail

Si sélectionné, le formulaire "Soumettre un ticket" affichera un "E-mail de confirmation" forçant le client à saisir deux fois l'adresse e-mail. Cela permet d'éviter les fautes de frappe dans les adresses e-mail.

 

Fermer la fenêtre

 

 

 

 

Ouvert uniquement

Si coché, l'e-mail "ID de ticket oublié" (e-mail envoyé pour rappeler aux clients leur numéro d'identification de suivi de ticket) ne listera que les tickets ouverts et ignorera les résolus.

 

Fermer la fenêtre

 

 

 

 

Formatage des e-mails

Détermine quels formats d'e-mail sont inclus lors de l'envoi d'e-mails. Les options incluent :

HTML avec alternative en texte brut à partir du modèle de texte brut (par défaut) - les e-mails seront envoyés au format HTML. Le contenu du modèle d'e-mail en texte brut Hesk sera utilisé pour le texte alternatif dans l'e-mail (multipart/alternative).
HTML avec une alternative en texte brut générée à partir du modèle HTML - les e-mails seront envoyés au format HTML. Le texte alternatif dans l'e-mail (multipart/alternative) sera généré en convertissant le modèle HTML en texte brut. Les modèles d'e-mails en texte brut Hesk seront ignorés.
HTML uniquement - les e-mails seront envoyés au format HTML uniquement sans multipart/alternative.
Texte brut uniquement - les e-mails seront envoyés en texte brut.
